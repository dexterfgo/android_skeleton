Installation Guide

Please install the AEVI SDK by unzipping the AEVI SDK Distribution to a chosen location on your development workstation.

Installing the Android Services and Sample Apps

You will first need to install two Android Apps called the ‘Payment Simulator’ and the ‘Device Simulator’ onto your development Tablet Device.

  • The ‘Payment Simulator’ is used to mimic the real Payment Application that is installed on the Albert,
  • The ‘Device Simulator’ is used to provide a simulation of the other services available on the real device.

Also there are two different versions of the ‘Device Simulator’ available:

  • The DeviceSimulator is for your development tablet (e.g. a Nexus 7) and the AEVI Emulator version 1.x.
  • The DeviceSimulator-emulator is used for the AEVI Emulator version 2.0 and later.

The Apps are installed onto your Android development Tablet Device using the Android Debug Bridge (ADB) and the command line.

Caution! The order of installation matters, you MUST install the two AEVI Simulators first. This is due to the security mechanism of Android where permissions are granted at install time. Some of the components expose permissions that are required by others, these permissions must therefore be known to Android prior to installation. If the requested permissions can not be found (i.e. when the app that exposes them was not installed) they are not granted in hindsight.

Please ensure your current working directory is set to the root of the AEVI SDK installation folder.

  1. To install the AEVI Device Simulator

    To install on a generic Tablet or AEVI Emulator Version 1.x:

     adb install android/simulators/DeviceSimulator-2.3.0.apk

    To install on the AEVI Emulator Version 2.0 and later:

     adb install android/simulators/DeviceSimulator-2.3.0-emulator.apk

  2. To install the AEVI Payment Simulator:

     adb install android/simulators/PaymentSimulator-2.3.0.apk

  3. To optionally install the AEVI Sample app:

     adb install samples/AeviSample-2.3.0.apk

  4. To optionally install the BarPos Example app:

     adb install samples/BarPosExample-2.3.0.apk

  5. To optionally install the To The Movies app:

     adb install samples/ToTheMovies-2.3.0.apk

  6. To optionally install the To The Movies Classic app:

     adb install samples/ToTheMoviesClassic-2.3.0.apk

Installing on the Albert developer device.

A signed version of the Payment Simulator and the AEVI sample apps is provided for the Albert developer device. These apks can be identified by the -albert suffix. Please see Albert Sample Apps section for the details.

Upgrading from a previous SDK Version

To upgrade from the previous release of the AEVI SDK, the sample apps, services and AEVI Simulator must be uninstalled from the tablet before commencing the upgrade. These apps can either be uninstalled by using the Settings > Apps screen or via the command line using ADB commands.

Uninstalling an AEVI SDK 2.x.x release using the Settings App

Open the Settings screen and click on the Apps entry. This should return a list of installed apps. The following apps must be uninstalled before upgrading;

  1. AEVI Device Simulator
  2. AEVI Payment Simulator
  3. AEVI Sample
  4. BarPosExample
  5. ToTheMovies

Click on each app entry to open the app properties screen and then click the Uninstall button to remove the app from the device.

Uninstalling an AEVI SDK 2.x.x release using ADB

Execute the following commands to uninstall any previous versions of the AEVI SDK apps

  1. AEVI Device Simulator

     adb uninstall com.aevi.simulator.device

  2. AEVI Payment Simulator

     adb uninstall com.aevi.simulator.payment

  3. Sample

     adb uninstall com.aevi.simpleexample

  4. BarPosExample

     adb uninstall com.aevi.barposexample

  5. ToTheMovies

     adb uninstall com.aevi.tothemovies

  6. ToTheMoviesClassic

     adb uninstall com.aevi.tothemovies.classic
