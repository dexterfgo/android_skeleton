Albert Introduction

Albert is a portable, multifunctional, interactive tablet based payment device. Based on a hardened version of the Android operating system, it combines the openness of this application oriented platform with the stringent security standards of a payment (EFTPOS) terminal. Albert enables businesses to install and use Android business and productivity apps, read cards, accept payments and print receipts, all within a single device.

Albert

Device Architecture

Albert is comprised of two core modules, the first is the Main Board the second the Security Module. All the components in Albert has been designed to be tamper resistant. If you have access to an Albert, do not attempt to open it as the tamper mechanism will trigger, therefore rendering it useless.

Main Board

The Main Board is similar to that of the main printed circuit board of an Android based Tablet. It contains the main processor (ARM based); memory; and interfaces to key peripherals such as Display, Wi-Fi, 3G Modem, Bluetooth, USB, and Sound. For the specification of Albert, please refer to the Device Specifications

This board also has some additional specific features:

  • USB connectivity to the integral Printer
  • A Secure Boot Module that verifies the integrity of the Operating System and Boot Files as part of the Boot Process
  • The Europay, MasterCard and Visa (EMV) kernel used to access Chip and Contactless Cards as part of the Payment process.

Security Module

The Security Module is a Secure Cryptographic device that provides the underlying cryptographic capabilities for taking payments.

The Security Module controls access to sensitive peripherals including:

  • Touch Screen interface
  • Chip Card Reader
  • Magstripe Reader
  • Near Field Communications (NFC) Reader

The touch screen interface is considered sensitive due to the fact that PIN Entry occurs on the touch display. Under normal operation, all touch events pass through the Security Module to the Main Board, however during PIN Entry mode, Touch Events are processed by the Security Module. The PIN entered by the Cardholder is encrypted and passed onto the Payment Host for verification. Alternatively, for offline CVM (cardholder verification method) the Secure Box will undertake PIN verification in conjunction with the Payment Card before encrypting the PIN and storing it offline or passing on the message to the Payment Host.
