App Development Lifecycle

This section provides a brief overview of how to develop an Android app for the AEVI platform and includes links to the full documentation. As the Albert Device is used in Point of Sale transactions, there many are important differences from developing a standard Android app.

Creating an AEVI App

Having installed the AEVI SDK, tried the samples and built a simple app; you are now ready to build a real app. This section provides you with a summary of the steps you need to follow:

  1. Ensure that your proposed app meets the policies and guidelines specified in the Developer Guidelines on the Developer Portal.
  2. Review the Albert Specifications, with particular reference to: the Screen Size, Resolution, Storage Sizes and Network Connectivity. See Albert Specifications.
  3. Familiarise yourself with the limitations and restrictions set out for your app, as described in the Platform Hardening & Restrictions document.
  4. Decided the app type, as defined in the Type Enforcement section and declare the type in the manifest file. Only types with ‘vaa_merchant’ have access to AEVI SDK API.
  5. For apps that require network connections you must to declare the Network access endpoints and connectivity requirements in the manifest file.
  6. Ensure that you only use the Supported Android Permissions in your app.
  7. Familiarise yourself with the different APIs provided with the AEVI SDK and remember to add the relevant AEVI Permissions to your manifest file.
  8. WebView Widgets must include the JavascriptInterface annotation on all functions exposed to JavaScript.
  9. Review whether the APIs you wish to use are compatible with the target bank or banks using the table in the API Compatibility section.
  10. Ensure you have used a Launcher App Icon Size of twice the normal size.
  11. Test your app on an appropriate Device (or Emulator). As we have said previously, the Google Nexus 7 (2012) is an excellent option.
  12. Test and debug your app using the AEVI emulator.
  13. Run on a test version of Albert. Your Bank may be able to offer you a test version of Albert. Please speak to them for further details.
  14. Once all this is working - you are done! Congratulations, you can submit your app to the Bank for review and deployment.

Testing your apps

We recognise that you may have formal testing processes that you apply to app development. However, this section provides guidance to suitable options for specific kinds of testing.

Using the Device and Payment Simulators

Installing the AEVI simulators on a generic tablet or the Android emulator provides a perfect method to undertake testing and validation in the development environment, encompassing both unit testing and debugging.

Running the AEVI simulators on a generic Android tablet provides an appropriate environment for this largely interactive activity.

Information on how to install the AEVI simulators and other services onto an Android device is described in the Installation Guide.

Using the AEVI Emulator

The AEVI Emulator is provided as part of the AEVI SDK and is an important step in verifying that your app will work on Albert. The emulator includes a version of the SELinux hardened image and so includes many of the restrictions that apply on the production Albert Device. This allows the following features to be tested that are representative of the production Albert:

  1. Type Enforcement
  2. Network Communications
  3. Secure Communications
  4. Supported Android Permissions

These features cannot be tested on a standard Android device or emulator and so it is very important that you test all your applications on the Albert device image before submitting your app for deployment.

The AEVI Emulator currently allows standard APK to be installed without declaring any application type. However applications without the declared application type will not be allowed on the production Albert Device as set out in the Type Enforcement section.

One of the advantages of the emulator is that can also be used for automated testing in a continuous integration environment.

For instructions on how to install the AEVI Emulator see the AEVI Emulator section.

Using a Test Version of Albert

Currently, there is no test version of Albert available. However, we are exploring options of how to make these available to Developers, including:

  1. Albert Sales
  2. Albert Loans
  3. Remote Access / Remote Control

Please watch this space for future announcements.
