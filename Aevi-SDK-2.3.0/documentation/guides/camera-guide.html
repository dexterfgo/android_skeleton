Using the Camera

The Camera has always been considered as one of the key peripherals on Albert. Even from the early concept phase, it was expected that App developers would have many innovative uses for the camera and therefore, the designers have been extremely careful about optimal placement for this peripheral. Another design constraint is the on camera placement is the overall security of the Device, specifically ensuring the privacy of the Cardholder, particularly during PIN entry.

This guide provides details on utilising the camera from your App and provides tips making best use of the device. The Guide assumes the reader has prior knowledge with the Android mechanisms for utilising the Camera. If this is not the case, the reader is requested to review the Android Guides and JavaDocs.

Camera App and Camera Intents

One of the key considerations on using the Camera on Albert is that the device does not ship with any Camera related Applications. The Intent based mechanism for capturing images or videos is not available.

App developers are required to use the Camera API directly. Further information is provided in the next section.

Building your own Camera Application

The official Android documentation provides a useful guide to using the Camera API. There are numerous other tutorials available on-line. This section is therefore limited to providing tips and tricks that the App Developer must be aware of when building an App that uses the Albert camera.

Camera Features

The key characteristics of the camera are summarised in the list below:

  • The device is based on the 8 mega-pixel Lite-On 11P2BA846B module.
  • It supports auto-focus from 10cm to infinity.
  • There are no additional APIs to access the Camera.
  • The Camera is configured as a rear facing camera. It performs no image manipulation such as image rotation or mirroring.

The Camera can be accessed by either calling the Camera.open() or Camera.open(0) static methods on the Android Camera API.

Enabling Auto Focus

In order to enable Auto Focus, please use the API method Camera.Parameters.setFocusMode(Camera.Parameters.FOCUS_MODE_CONTINUOUS_PICTURE) or Camera.Parameters.setFocusMode(Camera.Parameters.FOCUS_MODE_CONTINUOUS_VIDEO).

Using the Camera to scan Barcodes

One of the key uses App Developers have identified for the camera is barcode scanning. We have tested this feature using a number of Java based Barcode scanning libraries including the open source ZXing and the commercial SCANDIT libraries.

Note that should you decide a third party library, you may have to consider the rotation of any views as the placement of the Albert camera is optimised for a portrait based view.
