Mail Service

The Mail Service provides a straightforward and easy way to integrate e-mail features into your app. This section describes how-to get started and will walk the reader through sending a simple e-mail from their existing app.

Permissions

In order to interact with the Mail Service, each app needs to include the following permission in its Manifest file.

<uses-permission android:name="com.aevi.permission.MAIL_SERVICE" />

For more information about the available permissions, please see Manifest section.

Example usage

Constructing a message

Constructing a mail message can be done via the MailMessage class. After creating an empty message (new MailMessage()) the properties of the message such as the to address and subject can be set by calling methods on the MailMessage object.

Constructing a plain text email would be done like this :

new MailMessage()
  .addTo("info.nl@wincor-nixdorf.com")
  .setSubject("Plain test mail")
  .setPlainContent("This is a plain text mail message.\n \n Greetings \n \n The AEVI Team");

A message can either have HTML content or plain text content. Calling the setHtmlContent or setPlainContent will clear any existing content.

Attachments are added as byte arrays together with a name and a content type. Both the name and the content type are used for email applications. For the content type an internet media type should be used. After composing a mail the regular service mechanism can be used for sending the mail.

Below is an example for sending a mail with two attachments and a html message body

public void sendMail() {
  serviceProvider.connect(new AeviServiceConnectionCallback<MailService>() {
    @Override
    public void onConnect(MailService service) {
      try {
        service.send(createMessageWithAttachment());
      } catch (IOException e) {
        Log.e(TAG,"Error while trying to send a mail with attachments",e);
      }
    }
  });
}

private MailMessage createMessageWithAttachment() throws IOException {
  return new MailMessage()
      .addTo(getValue(R.id.emailTo))
      .setSubject("Attachment test mail")
      .setHtmlContent("<html>" +
          "<body>" +
          "<h1>Example</h1>" +
          "<p>This is a html mail message with an attachment.</p>" +
          "<p>Greetings</p>" +
          "<p>The AEVI Team</p>" +
          "</body>" +
          "</html>")
      .addAttachment(createAttachment("cba_icon.png", R.drawable.cbalogocolor, "image/png"))
      .addAttachment(createAttachment("wincor_icon.png", R.raw.wincor_icon, "image/png"));
}

private MailMessageAttachment createAttachment(String name, int id, String type) throws IOException {
  return new MailMessageAttachment(name, readResource(id), type);
}

private byte[] readResource(int id) throws IOException {
  InputStream inputStream = getResources().openRawResource(id);
  byte[] result = new byte[inputStream.available()];
  inputStream.read(result);
  return result;
}

Simulation of the Mail API

The simulated environment does not send out emails but instead saves the content of the email to disk so that it can be later displayed using the AEVI Device Simulator. Only the name of any attachments are recorded and it is not possible to view the contents of any attachments when using the Device Simulator. However when the Mail API is used on the Albert the email is sent out immediately with all the attachments.

On Albert, Merchants are required to configure their eMail credentials as part of the Device setup. The email service uses the Simple Mail Transfer Protocol (SMTP) to send emails through the Merchantâ€™s preferred provider.

Note: On the AEVI Device Simulator the user must manually delete each test email after it has been sent.

Limitations

  1. At the moment there is no feedback mechanism for reporting errors that occur while sending an email from an app. These errors are however logged.
  2. The total size of the email (text + attachments) can not be larger than 1Mb.
